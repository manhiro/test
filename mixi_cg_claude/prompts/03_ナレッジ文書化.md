# 社内ナレッジ文書化 プロンプト集

ベテランの暗黙知をチームの共有ナレッジに変換するためのプロンプト集。

---

## 口頭・メモ → マニュアル変換

### 作業手順のマニュアル化
```
以下の作業手順メモをチーム向けのマニュアルに変換してください。
読者は [経験レベル: 新人 / 中堅] です。

【元メモ】
[箇条書きや口頭メモをそのまま貼る]

出力形式：
# [作業名] マニュアル

## 目的
## 前提条件・必要ツール
## 手順
  Step 1: ...（スクリーンショット挿入箇所: ○○の画面）
  Step 2: ...
## よくあるミスと対処法
## 応用Tips
## 最終更新: [日付] / 担当: [名前]
```

### 会議・レビューの内容をナレッジ化
```
以下のレビュー会議のメモから、チームナレッジとして
残すべき情報を抽出してください。

【会議メモ】
[議事録や走り書きをそのまま貼る]

抽出してほしいもの：
1. 再利用できる技術的な判断基準
2. 今後のプロジェクトに適用すべきルール
3. 解決した問題とその解決策（FAQ候補）
4. まだ未解決で調査が必要な課題
```

## FAQ・トラブルシューティング集

### エラー・問題の解決策をFAQ化
```
以下のトラブルとその解決策をFAQ形式でまとめてください。

【トラブル内容】
ツール: [Blender / Houdini / UE5 等]
症状: [何が起きたか]
環境: [OS・バージョン等]
解決方法: [どうやって直したか]

出力形式：
Q: [症状を1行で表現]
A: 
  原因: ...
  解決手順:
    1. ...
    2. ...
  予防策: ...
  参考リンク: （あれば）
```

### チーム内暗黙知のインタビュー → 文書化
```
私はCG映像制作のベテランです。
あなたはナレッジマネージャーとして、
私の暗黙知を引き出すインタビューをしてください。

テーマ: [例：ライティングの判断基準 / レンダリング最適化のコツ]

私の回答をもとに、新人でも理解できるドキュメントを
インタビュー後にまとめてください。

※インタビューは1問ずつ聞いてください。
```

## オンボーディング資料

### 新人向けチームガイドの作成
```
以下の情報をもとに、新しくチームに加わったCGアーティスト向けの
オンボーディングガイドを作ってください。

チームの特徴: [説明]
使用ツール一覧: [ツール名とバージョン]
よくある最初のつまずきポイント: [箇条書きで]
チームのルール・文化: [箇条書きで]

含めてほしいセクション：
1. チームの仕事の流れ（フロー図的に）
2. 最初の1週間でやること
3. 使うツールとアカウント取得方法
4. 質問・相談の方法（Slackチャンネル等）
5. やってはいけないこと（ファイル管理等）
```

### スキルマップの作成
```
以下のチームメンバーのスキルを整理して、
スキルマップを作ってください。

メンバー情報:
[名前]: [得意なこと・ツール・経験年数]
（複数人分貼る）

出力してほしいもの：
1. スキルマップ表（ツール×メンバー、習熟度を○△×で）
2. チームとして手薄なスキル領域
3. 育成・採用の優先度提案
```
