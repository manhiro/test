# 制作効率化 プロンプト集

スクリプト生成・シェーダー作成・パイプライン設計など、技術的な制作業務をClaudeで加速するプロンプト集。

## Pythonスクリプト生成

### Blender Pythonスクリプト
```
Blenderで以下の作業を自動化するPythonスクリプトを書いてください。

やりたいこと: [例：選択したオブジェクト全てのマテリアルを
              指定フォルダにエクスポートする]
Blenderバージョン: [4.x]
注意事項: [あれば]

コードには以下を含めてください：
- 処理の概要コメント
- エラーハンドリング
- 使い方の説明（スクリプトエディタでの実行方法）
```

### Houdini Python / HScript
```
Houdiniで以下の処理をするスクリプトを書いてください。

やりたいこと: [説明]
スクリプト種別: [Python / HScript / VEX]
ノードの種類: [SOP / DOP / CHOP 等]

コードと一緒に、ノード構成の説明も教えてください。
```

### Maya MEL / Python

```
やりたいこと: [説明]
対象: [シーン内の全メッシュ / 選択オブジェクト 等]
```

## シェーダー・マテリアル

### GLSL / HLSL シェーダー生成
```
以下のビジュアルエフェクトを実現するシェーダーを書いてください。

エフェクト: [例：ホログラムのスキャンライン効果]
ターゲット: [Unity HLSL / Unreal HLSL / OpenGL GLSL]
入力パラメーター: [色・速度・強度等、調整したいもの]

コードには：
- 各パラメーターの説明
- マテリアルエディタでの設定方法
- 負荷の目安
を含めてください。
```

### Houdini VEX スニペット
```
以下の処理をするHoudini VEXコードを書いてください。

処理内容: [説明]

入力ジオメトリ: [ポイント数の目安・構造]

よくあるVEXの落とし穴があれば注意事項も教えてください。
```

## パイプライン・ワークフロー設計

### レンダーパイプラインの最適化提案
```
現在のレンダーパイプラインを最適化したいです。

現状:
- ソフト: [Blender / Houdini / Maya + Arnold 等]
- レンダーファーム: [ローカル / クラウド / 社内farm]
- 平均レンダー時間: [1フレームあたり XX分]
- 主なボトルネック: [説明]
- チーム規模: [XX人]

最適化できる箇所と、優先度の高い改善策を提案してください。
```

### ファイル命名規則・ディレクトリ構造の設計
```
CGプロジェクトのファイル管理ルールを設計してください。

プロジェクト規模: [1人 / 5人 / 20人チーム]
使用ソフト: [ツール一覧]
プロジェクト期間: [XX ヶ月]
バージョン管理: [Git / SVN / 独自 / なし]

出力してほしいもの：
1. ディレクトリ構造（ツリー形式）
2. ファイル命名規則（例と理由）
3. バージョン・バリアント管理の方針
4. アーカイブルール
```
