## 事例詳細

### 事例1: キックオフ前のビジュアル方針の言語化

**状況:** 新プロジェクト開始。クライアントから「こういう感じで」と
参考画像が数枚届いただけで、具体的な方針が不明確。

**Claudeの使い方:**
```
以下の参考作品のビジュアルスタイルを分析して、
制作チーム向けのビジュアル方針ドキュメントを作ってください。

参考作品: [作品名1], [作品名2], [作品名3]
プロジェクト概要: [説明]
避けたい方向性: [説明]
```

**得られたもの:** ライティング・カラー・カメラワークの
具体的なガイドラインが30分で完成。チーム全員の認識が統一。

---

### 事例2: Houdini新バージョンの機能把握

**状況:** Houdini 20がリリース。何が変わったか調べる時間がない。

**Claudeの使い方:**
```
Houdini 20の新機能のうち、VFXシミュレーションと
プロシージャルモデリングの現場で重要な変更点を
優先度順にまとめてください。
各機能に「移行の手間」も教えてください。
```

**得られたもの:** 優先度付きの新機能リスト。
「今すぐ使う / 後回しでいい」の判断ができた。

---

### 事例3: レンダリング最適化スクリプトの作成

**状況:** シーン内の全オブジェクトのサブディビジョンレベルを
レンダリング用に一括変更したいが、手動では100オブジェクトある。

**Claudeの使い方:**
```
Blenderで選択した全オブジェクトのサブディビジョンモディファイアの
レンダーレベルを指定値に一括変更するPythonスクリプトを書いてください。
対象Blenderバージョン: 4.x
```

**得られたもの:** 動作するスクリプトが10分で完成。
手作業2時間 → スクリプト実行5秒に短縮。

---

### 事例4: ベテランの手順書作成

**状況:** 先輩のライティング手順を新人に伝える必要があるが、
先輩は忙しくて詳しいドキュメントを書く時間がない。

**Claudeの使い方（先輩との会話形式）:**
```
あなたはナレッジマネージャーです。
私（ベテランライター）の手順を新人向けマニュアルにするため、
インタビューしてください。
テーマ：スタジオライティングのセットアップ手順
1問ずつ聞いてください。
```

**得られたもの:** 30分のインタビューで、
新人でも再現できる詳細マニュアルが完成。

---

### 事例5: クライアントレビューコメントの整理

**状況:** クライアントから20項目のフィードバックが届いたが、
「ここ直して」「あそこも気になる」と混在していて優先度が不明。

**Claudeの使い方:**
```
以下のクライアントフィードバックを
「必須修正・推奨修正・次バージョン検討」に分類して、
作業タスクとして整理してください。
[フィードバックをそのまま貼る]
```

**得られたもの:** 優先度付きのタスクリスト。
その日の作業範囲が即座に決まった。

## Claude活用の注意点（チーム共通ルール）

### やっていいこと
- 技術調査・情報収集
- ドキュメント・マニュアル作成
- スクリプト・コードのドラフト生成（必ずレビューして使う）
- アイデア出し・ブレスト
- 議事録・メモの整理

### 注意が必要なこと
- **社外秘情報は入力しない**: クライアント名・未発表タイトル・契約情報等
- **生成コードは必ず動作確認**: そのまま本番環境に適用しない
- **ハルシネーション注意**: 技術仕様・バージョン情報は公式ドキュメントで確認

### Claudeが苦手なこと
- 最新バージョンの詳細仕様（学習データに含まれていない場合）
- 社内固有のツール・パイプラインの詳細
- 実際のレンダリング・ビジュアルの判断（テキストベースのみ）
