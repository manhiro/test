# リファレンス収集・整理 プロンプト集

制作に入る前のリファレンス収集・ムードボード作成・ビジュアル方針の言語化を効率化するプロンプト集。

---

## プロジェクト立ち上げ時

### ビジュアル方針の言語化
```
以下のプロジェクトのビジュアル方針をまとめてください。

プロジェクト名: [名前]
ジャンル: [ゲームトレーラー / アニメ / CM / ショートフィルム 等]
ターゲット: [年齢層・趣味嗜好]
参考にしたい作品: [タイトル1, タイトル2, ...]
避けたいスタイル: [具体例]

出力してほしいこと：
1. ライティング方針（時間帯・光源の性質・コントラスト）
2. カラーパレット（主色・補色・禁止色）
3. カメラワーク（焦点距離・動きの速度感・被写界深度）
4. テクスチャ・マテリアル感（リアル寄り / スタイライズ度）
5. NGリスト（このプロジェクトで絶対やらないこと）
```

### 競合リファレンスの差別化分析
```
以下の競合作品と差別化したCGビジュアルを作るために、
各作品の「強み」と「やり尽くされている表現」を分析してください。

競合作品:
1. [タイトル / URL]
2. [タイトル / URL]
3. [タイトル / URL]

分析の観点:
- ビジュアルスタイルの特徴
- 技術的な見どころ
- 視聴者に与えている印象
- 飽和しつつある表現（差別化ポイント）
```

## シーン・カット別リファレンス整理

### シーンのムードを言語化して検索キーワードを生成
```
以下のシーンのムードに合うリファレンス画像を探したいです。
ArtStation・Pinterest・Google画像検索で使える
英語検索キーワードを10〜15個生成してください。

シーンの説明:
[例：夕暮れの廃墟、光が差し込む、孤独感、SF的な雰囲気]

あわせて、このシーンに合う映画・ゲーム・アーティスト名も
3〜5つ挙げてください。
```

### ショットリスト → リファレンス方針変換
```
以下のショットリストに対して、各カットの
リファレンス収集方針を提案してください。

ショットリスト:
[ショットリストをそのまま貼る]

各ショットに対して：
- 参考にすべき映画・ゲームシーン（具体的に）
- ArtStation検索キーワード（英語）
- 注意すべき権利・著作権の問題点
```

## スタイルガイド・バイブルの作成

### ルックデベロップメント用スタイルガイド
```
以下の情報をもとに、チーム共有用のルックデベロップメント
スタイルガイドのドラフトを作ってください。

プロジェクト概要: [説明]
決定済みのビジュアル方針: [箇条書きで貼る]
使用ツール: [Blender / Houdini / Maya / Nuke 等]

含めてほしいセクション：
1. カラーマネジメント設定（ACES / sRGB 等）
2. ライティングの基準値
3. マテリアル命名規則
4. レンダリング設定の標準値
5. コンポジット・カラグレの基本方針
```

### 過去プロジェクトからの教訓抽出
```
以下のプロジェクトの振り返りメモから、
次回以降に活かせるリファレンス・ルール集を作ってください。

振り返りメモ:
[箇条書きや議事録をそのまま貼る]

出力してほしいこと：
1. 「このスタイルは正解だった」ベストプラクティス
2. 「これは二度とやらない」アンチパターン
3. 次プロジェクトへの申し送り事項
```
