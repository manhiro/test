CLAUDE.md とは？

Claude Code への「チーム説明書」——一度書けば毎回読んでくれる
💡 たとえるなら
新しいアルバイトスタッフに渡す「業務マニュアル」のようなものです。

毎日「うちのチームはHoudiniを使っていて…」と説明するのは大変ですよね。
CLAUDE.md に書いておけば、Claude は起動するたびにそれを読んで、
あなたのチームの前提を把握した状態で会話を始めます。

どう動くの？

📁 プロジェクトフォルダで
claude を起動
→
📄 CLAUDE.md を
自動で読み込む
→
💬 チームの前提を知った
状態で会話スタート

CLAUDE.md がなければ、Claude は「このフォルダが何のプロジェクトか」を毎回ゼロから理解しようとします。CLAUDE.md を置くだけで、初回からコンテキストが揃った状態になります。

テンプレート：コピペで使えます

# プロジェクト概要 MIXI CG映像チームのプロジェクト。 商業CG映像の制作を行っている。 # 使用ツール - Houdini 20.5（シミュレーション・VFX） - Unreal Engine 5.4（リアルタイムレンダリング） - After Effects 2024（コンポジット） - Blender 4.x（モデリング補助） # チーム構成 CG アーティスト × 2、ディレクター × 1 # ファイル命名規則 assets/[category]/[name]_[type]_v001.ext 例: assets/textures/char_skin_diffuse_v001.png # 返答スタイルの希望 - 専門用語には日本語の説明を必ず添えること - コードには使い方のコメントを入れること - 長い説明より、まず結論を先に書くこと

書けることの例

✅ 書くと効果的なこと

プロジェクトの目的・使用ツールとバージョン・チーム構成・ファイル命名規則・よく使う用語の定義・返答スタイルの希望（箇条書きで、など）

💡 役割別に複数作っても OK

プロジェクトごとに CLAUDE.md を作成することで、フォルダを移動するだけで Claude の「人格」が切り替わります。

📌
CLAUDE.md は普通のテキストファイルです。メモ帳・VS Code・どんなエディタでも編集できます。Markdown 記法（# 見出し や - リスト）が使えます。
✅
まず 5 行だけ書いてみましょう。プロジェクト名・ツール名・チーム人数。それだけでも Claude の返答の質が大きく変わります。
