Section 01

Claude Code って
何ができるの？

AIに日本語で話しかけるだけで、面倒な作業を肩代わりしてくれるツールです。プログラミングの知識はいりません。

📁
フォルダを丸ごと理解
プロジェクト全体を読んで、何がどこにあるか把握。整理・リネームの提案まで。
📝
ドキュメントを自動作成
シーン設定・シェーダー仕様・作業ログを、そのまま読みやすい文書に変換。
⚙️
スクリプトを生成・修正
「こういう処理をしたい」と伝えるだけでVEX・Python・MELを書いてくれる。
🔍
技術調査・まとめ
Blender / UE5 の新機能やベストプラクティスを調べ、チーム向けにまとめる。
🔁
繰り返し作業を自動化
毎週同じ形式のレポート、アセットのチェック作業——一度設定すれば後は任せるだけ。
💬
会話しながら作業
「もう少しシンプルに」「英語でも書いて」と追加指示も自然な日本語でOK。
💡
Claude Code はターミナル?（黒い画面）で動きますが、操作は会話形式です。コマンドをほとんど覚える必要はありません。
Section 02

3ステップで
セットアップ完了

Windows でも Mac でも同じ手順です。所要時間：約 10 分。

Step 1 — Node.js をインストール

Claude Code を動かすために必要なソフトウェアです。Node.js とは？?

🌐
nodejs.org にアクセスして LTS（推奨版） をダウンロード→インストール。「次へ」を押し続けるだけでOKです。

インストール後、ターミナルで確認：

PS>$ node --version
✅
v18.x.x 以上が表示されれば成功です。
Step 2 — Claude Code をインストール

ターミナルで以下を実行します。コピーボタンで貼り付けるだけ。

PS>$ npm install -g @anthropic-ai/claude-code
⚠️
エラーが出た場合、会社のプロキシ設定が必要な場合があります。情シスに確認してみてください。
PS>$ claude --version
✅
バージョン番号が表示されれば成功です。
Step 3 — 認証する

2 つの方法があります。APIキーとは？?

★ おすすめ
claude.ai Pro/Max プラン
ブラウザでログインするだけ。APIキー不要で最も簡単。
API KEY
APIキーで認証
従量課金型。console.anthropic.com でキーを発行。
PS>$ claude
✅
ブラウザが開いてログイン画面が表示されます。claude.ai アカウントでログインすれば完了です。
Section 03

最初のプロンプトを
入力してみよう

プロンプト?とは AI への指示文のこと。日本語で自然に話しかけて OK です。

作業したいフォルダで起動：

PS> cd C:\Users\YourName\Documents\MyProject
PS> claude
$ cd ~/Documents/MyProject
$ claude

起動したら日本語で話しかけるだけ：

You › このフォルダの構成を説明してください
Claude › このプロジェクトには以下のファイルが含まれています…
💡
終了は /exit または Ctrl+C を押してください。

Section 05

知っておくと差がつく
Tips

セットアップ後に押さえておきたい、実務で役立つテクニック集。

💡 プロンプトのコツ
  • 結論から書く — 「〇〇して。対象は～、出力は～」の順が精度高い
  • エラーはそのまま貼る — コピペして「このエラーを直して」だけでOK
  • 出力形式を指定する — 「箇条書きで」「3行以内で」「表形式で」
  • 逆に聞く — 「これを実現するためにどんな情報が必要？」
⚡ 時短テクニック
  • ↑キー で前のプロンプトを再呼び出し
  • /compact で長くなった会話を要約してリセット（精度回復）
  • フォルダごと渡す — 「このフォルダの全Pythonファイルを読んで」でOK
  • よく使う指示は .claude/commands/ にテンプレ保存 → 詳細
🚫 よくある失敗と対処
  • 的外れな回答 → 背景・条件が不足。もっと具体的に伝える
  • 処理が止まる → Ctrl+C で中断、タスクを分割して再依頼
  • 同じ間違いを繰り返す → /clear でリセット or CLAUDE.md に「〇〇はしないで」と明記
  • 返答が長すぎる → 「3行で」「箇条書き5点で」と制限する
🚀 上級テクニック
  • Sub-agents — 「並列で各ファイルを分析して」と指示するだけで自動分担 → 詳細
  • Hooks — 「ファイル保存後に自動チェック」などの自動化
  • Slack連携 — Slackを読んで議事録を自動まとめ → 設定方法
  • Notion連携 — ドキュメントを読んでレポート自動生成 → 設定方法
Section 06

もっと使いやすく
カスタマイズする

CLAUDE.md? をプロジェクトフォルダに置くと、Claudeが毎回チームの前提を知った状態でスタートします。

プロジェクトフォルダに CLAUDE.md を作成して書くだけ：

# プロジェクト概要 MIXI CG映像プロジェクト。Houdini + UE5で制作。 # 使用ツール Houdini 20.5 / UE5.4 / After Effects 2024 # ファイル命名規則 assets/textures/[asset]_[type]_v001.png # 返答スタイルの希望 - 専門用語には日本語の説明を添えること - コードには使い方コメントを入れること
💡
このファイルがあるフォルダで claude を起動すると、毎回この内容を読んでから会話が始まります。

よく使うコマンド一覧

コマンド説明
/help使えるコマンドの一覧を表示
/clear会話履歴をクリアして新しいセッションを開始
/compact長くなった会話を要約してリセット（コスト削減・精度回復）
/exitClaude Code を終了
Ctrl + C現在の処理を中断
↑ キー前に入力したプロンプトを再呼び出し
