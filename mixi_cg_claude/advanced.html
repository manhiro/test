Slash commands — プロンプトのテンプレ化
よく使う指示を1コマンドで呼び出す。
中級 ▼
.claude/commands/ フォルダにMarkdownファイルを置くだけ。毎回同じプロンプトを打たなくて済みます。

例：/vex で VEX コード生成、/doc でドキュメント作成、/report で週報生成。
例：.claude/commands/vex.md を作成
# VEX コード生成 $ARGUMENTS に書かれた処理をHoudiniのVEXで実装してください。 - アトリビュート名はキャメルケース - 各行にコメントを入れること - 使い方の例も添えること
→ /vex Pointをランダムに削除する処理 で呼び出せる

Hooks — 自動トリガー
ファイル保存・コミット前に自動でClaudeが動く。
上級 ▼
.claude/settings.json に hooks を設定すると、特定のイベントが起きたとき自動でスクリプトやコマンドを実行できます。

❌ Hooksなし
ファイルを保存 → 手動でClaudeに「チェックして」と依頼
✅ Hooksあり
ファイルを保存 → 自動でClaudeが構文チェック・コメント補完
{ "hooks": { "PostToolUse": [{ "matcher": "Write", "hooks": [{ "type": "command", "command": "echo '保存しました'" }] }] } }

Sub-agents の使い方

Sub-agents を使うために特別な設定は不要です。Claude Code は指示に応じて自動的にサブエージェントを立ち上げます。指示の書き方だけが変わります。

自動で Sub-agents が動く指示パターン

  • ①
    大量ファイルの並列分析
    「このプロジェクトの全 Python ファイルをそれぞれ並列で分析して、問題点をリストアップして」
  • ②
    複数の成果物を同時生成
    「このHoudiniプロジェクトの技術仕様書・操作マニュアル・引き継ぎドキュメントを同時に作成して」
  • ③
    複雑なリサーチ＋実装
    「UE5のNanitaの最適設定を調べながら、同時にこのシーンのLOD設定を見直して」
❌ 逐次処理（遅い）
「ファイルAを分析して。終わったらBを、終わったらCを…」
✅ 並列処理（速い）
「A・B・Cを並列で分析して、結果をまとめてレポートにして」
💡
「並列で」「同時に」「それぞれ」という言葉を含めるだけで、Claude Code は自動的に Sub-agents を活用した処理を選択します。

Slash commands のセットアップ

CG映像チーム向けのコマンドをそのまま使えるテンプレートです。

ディレクトリ構成

プロジェクトフォルダ/ └── .claude/ ├── CLAUDE.md └── commands/ ├── vex.md # /vex → VEXコード生成 ├── doc.md # /doc → ドキュメント作成 ├── report.md # /report → 週報生成 └── review.md # /review → コードレビュー

📄 commands/doc.md — ドキュメント自動生成

# ドキュメント生成 このプロジェクトを読んで、以下の形式でドキュメントを作成してください。 ## 対象 $ARGUMENTS（指定がなければフォルダ全体） ## 出力形式 - 概要（2〜3行） - ファイル構成 - 主要な機能・設定 - 注意点・既知の問題 ## スタイル - 読者はCG制作者（非エンジニア）を想定 - 専門用語には説明を添えること
使い方：/doc このシーンのセットアップ

⚡ commands/vex.md — VEXコード生成

# VEX コード生成 以下の処理をHoudini VEXで実装してください。 ## やりたいこと $ARGUMENTS ## 要件 - Wrangle ノードに貼るだけで動くコード - 各行に日本語コメントを入れること - アトリビュート名と使い方を冒頭に書くこと - 使用例（どのノードに貼るか）を最後に書くこと
使い方：/vex Pointをheightに応じてRGB着色する

📊 commands/report.md — 週報自動生成

# 週報生成 このフォルダの作業ログ・メモを読んで、週報を作成してください。 ## 出力形式 **今週の成果** - （箇条書き3〜5点） **来週の予定** - （箇条書き3〜5点） **課題・ブロッカー** - （あれば記載） ## 注意 - ディレクターが読むことを想定すること
使い方：/report（引数なしでOK）
✅
コマンドファイルは普通のテキストファイルです。メモ帳でも編集できます。$ARGUMENTS が、コマンド実行時に打った追加テキストに置き換わります。
